Understanding Django Templates

In this lesson, we will explore how Django templates work. Django templates are a powerful tool for rendering HTML content dynamically, allowing you to build dynamic web pages with Python variables, control structures, and more.

Step 1: Introduction to Django Templates

Django templates allow you to separate your HTML from your Python code. Templates use a simple syntax to embed dynamic content into HTML files. Let’s start by creating a basic template.

{% verbatim %} 
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}My Site{% endblock %}</title>
</head>
<body>
    <header>My Header</header>
    {% block content %}{% endblock %}
    <footer>My Footer</footer>
</body>
</html>
{% endverbatim %}

In this template, {{ name }} is a placeholder for dynamic content that will be replaced by the value passed from the view.

Step 2: Creating a View to Render the Template

To render this template, you need to create a view in Django. Here’s how you can define a view that passes data to the template:


from django.shortcuts import render

def hello_view(request):
    context = {'name': 'World'}
    return render(request, 'hello.html', context)

This view creates a dictionary context with a key name and passes it to the hello.html template.

Step 3: Creating the URL Pattern

Next, you need to map a URL to this view. Add the following URL pattern to your app’s urls.py:

from django.urls import path
from . import views

urlpatterns = [
    path('hello/', views.hello_view, name='hello'),
]

Now, when you navigate to http://127.0.0.1:8000/hello/, you will see the rendered template with the name "World".

Step 4: Using Template Tags and Filters

Django templates support a variety of template tags and filters to manipulate and format data. For example:

{% verbatim %}{{ name|upper }}{% endverbatim %}

This will convert the name to uppercase. You can also use control structures like loops and conditionals:

{% verbatim %}{% if name %}
    <h1>Hello, {{ name }}!</h1>
{% else %}
    <h1>Hello, World!</h1>
{% endif %}
{% endverbatim %}

Step 5: Template Inheritance

Django templates support inheritance, allowing you to create a base template and extend it. Here’s an example of a base template:

{% verbatim %}
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}My Site{% endblock %}</title>
</head>
<body>
    <header>My Header</header>
    {% block content %}{% endblock %}
    <footer>My Footer</footer>
</body>
</html>
{% endverbatim %}

To extend this base template, create a child template:

{% verbatim %} 

{% extends "base.html" %}

{% block title %}Home{% endblock %}

{% block content %}
    <h1>Welcome to the Home Page!</h1>
{% endblock %}
{% endverbatim %}

This will insert the content of the child template into the corresponding blocks of the base template.

Best Practices for Django Templates

  • Keep your templates organized and avoid excessive logic in templates.
  • Use template inheritance to maintain a consistent layout across pages.
  • Leverage template tags and filters to format and manipulate data effectively.
  • Ensure templates are responsive and adhere to best practices for HTML and CSS.
